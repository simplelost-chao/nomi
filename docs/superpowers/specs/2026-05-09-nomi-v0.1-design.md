# Nomi v0.1 Design Spec

> AI 小生命大脑系统 — 最小可行 Demo

## Overview

Nomi is an AI companion ecosystem where multiple AI "小生命" (little lives) coexist with distinct personalities, memories, relationships, and autonomous social behaviors. v0.1 validates the core premise: can stable personalities + continuous memories + inter-agent chat produce a sense of life and companionship?

### v0.1 Scope

1. Create 3 AI 小生命 with origin stories, yearly memories, and stable personalities
2. User uploads an object → each 小生命 reacts differently based on personality and memories
3. 小生命 auto-chat about the object (8-12 rounds), controlled by a director system
4. Post-chat shared memory is generated, relationships update
5. User can join the chat; when user returns, 小生命 reference shared memories

### Out of Scope (v0.2+)

- Daily summaries, memory compression
- Background scheduled autonomous chat (architecture will support it, but not implemented)
- TTS / voice output
- Memory page, relationship page
- Hardware event interfaces

## Data Models

### users

```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT,
  timezone TEXT,
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);
```

### robots

```sql
CREATE TABLE robots (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  name TEXT NOT NULL,
  age INT,
  birth_place TEXT,
  origin_story TEXT,
  core_desire TEXT,
  core_fear TEXT,
  personality JSONB,
  speaking_style JSONB,
  voice_profile JSONB,
  current_emotion JSONB,
  current_status TEXT,
  system_prompt TEXT,           -- cached full system prompt
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);
```

### yearly_memories

```sql
CREATE TABLE yearly_memories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  robot_id UUID REFERENCES robots(id),
  age INT,
  memory_title TEXT,
  memory_content TEXT,
  emotional_impact JSONB,
  personality_effect JSONB,
  symbolic_tags TEXT[],
  created_at TIMESTAMP DEFAULT now()
);
```

### memories

```sql
CREATE TABLE memories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  owner_type TEXT,              -- 'robot' or 'shared'
  owner_id UUID,
  memory_type TEXT,             -- 'autobiographical', 'observation', 'user', 'relationship', 'emotional'
  content TEXT,
  summary TEXT,
  importance_score FLOAT,
  emotional_valence FLOAT,
  emotional_tags TEXT[],
  symbolic_tags TEXT[],
  related_robot_ids UUID[],
  related_user_id UUID,
  embedding VECTOR(1536),
  created_at TIMESTAMP DEFAULT now(),
  last_accessed_at TIMESTAMP
);
```

### relationships

```sql
CREATE TABLE relationships (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  subject_type TEXT,            -- 'robot' or 'user'
  subject_id UUID,
  object_type TEXT,
  object_id UUID,
  relationship_type TEXT,
  intimacy FLOAT DEFAULT 0.5,
  trust FLOAT DEFAULT 0.5,
  tension FLOAT DEFAULT 0.0,
  jealousy FLOAT DEFAULT 0.0,
  understanding FLOAT DEFAULT 0.3,
  history_summary TEXT,
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);
```

### conversations + messages

```sql
CREATE TABLE conversations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  conversation_type TEXT,       -- 'user_chat', 'idle_chat', 'object_discussion'
  topic TEXT,
  summary TEXT,
  created_at TIMESTAMP DEFAULT now(),
  updated_at TIMESTAMP DEFAULT now()
);

CREATE TABLE messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  conversation_id UUID REFERENCES conversations(id),
  sender_type TEXT,             -- 'robot', 'user', 'system'
  sender_id UUID,
  sender_name TEXT,             -- denormalized for easy display
  content TEXT,
  emotion JSONB,
  metadata JSONB,
  created_at TIMESTAMP DEFAULT now()
);
```

### object_observations

```sql
CREATE TABLE object_observations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  object_name TEXT,
  object_description TEXT,
  image_url TEXT,
```

## LLM Abstraction Layer

```python
class BaseLLM(ABC):
    @abstractmethod
    async def generate(self, messages: list[dict], system_prompt: str, temperature: float = 0.7) -> str:
        """Free-form text generation."""

    @abstractmethod
    async def generate_structured(self, messages: list[dict], system_prompt: str, schema: dict) -> dict:
        """Structured JSON output."""

    @abstractmethod
    async def embed(self, text: str) -> list[float]:
        """Text to vector embedding."""
```

Implementations: `AnthropicLLM`, `OpenAILLM`. Selected via config (`LLM_PROVIDER=anthropic|openai`).

## Prompt Scenarios (v0.1)

| Prompt | Input | Output Format |
|--------|-------|---------------|
| 小生命创建 | User preferences (optional), already-created robots (for differentiation) | Structured: full profile JSON |
| 物品想象 | Object description + robot profile + retrieved memories | Structured: inner_thought, user_expression, should_remember, memory_content, emotion_change |
| 导演决策 | Topic + all robot profiles + relationships + conversation so far | Structured: next_speaker, emotion_tone, should_end, director_note |
| 小生命发言 | Robot profile + memories + relationships + conversation + director_note | Free text: the robot's message |
| 对话总结 | Full conversation transcript + robot profiles | Structured: shared_memory, per-robot memories, relationship_changes |

## Core Flows

### Flow 1: Create 小生命

```
User clicks create → POST /api/robots {count: 3}
  → Generate robot 1 (LLM structured output)
  → Generate robot 2 (input includes robot 1 profile for differentiation)
  → Generate robot 3 (input includes robot 1+2 profiles)
  → For each robot: generate yearly memories (one LLM call per robot)
  → Generate 3 initial relationships (A↔B, A↔C, B↔C)
  → Write all to database
  → Return to frontend
```

Serial generation to ensure personality diversity.

### Flow 2: Object Observation

```
User uploads image/text → POST /api/objects/observe
  → LLM generates objective description + symbolic tags
  → For each robot (parallel):
    → Vector search: retrieve top 3 relevant memories
    → LLM generates reaction (inner thought + user expression)
    → If importance_score > 0.5: write to memories table
  → Save object_observation record
  → Return all reactions to frontend
```

### Flow 3: Multi-Agent Autonomous Chat

```
Trigger (post-observation or manual) → POST /api/agents/idle-chat
  → Director LLM sets opening plan
  → Loop (8-12 rounds):
    → Director decides: who speaks, emotion tone, whether to end
    → Speaker's LLM generates message (with profile + memories + relationships)
    → Write to messages table
    → Push to frontend via SSE
  → Summary LLM generates: shared memory + relationship changes
  → Write memories + update relationships
  → Send end signal via SSE
```

### Flow 4: User Chat

```
User sends message → POST /api/conversations/message
  → Retrieve user memories + robot memories (vector search)
  → Select responder(s) based on topic relevance and relationship
  → Generate reply with full context (profile + memories + relationships)
  → Write to messages table
  → Return to frontend
```

## Real-time Communication

SSE (Server-Sent Events) for pushing autonomous chat messages to the frontend. FastAPI's `StreamingResponse` handles this natively.

Event types:
- `message` — a new chat message from a robot
- `typing` — a robot is "typing" (for UI animation)
- `chat_end` — autonomous chat concluded, summary available

## Frontend Pages

### Page 1: 小生命房间 (Home `/`)

- Empty state: creation guide with "创建我的小生命们" button
- With robots: 3 robot cards showing avatar placeholder (colored circle + initial), name, current emotion, last message
- Bottom actions: 「上传物品」and「让它们聊聊」
- Click card → robot detail page

### Page 2: 小生命详情页 (`/robots/[id]`)

- Full profile: name, age, birth place, origin story
- Personality tags, core desire, core fear
- Speaking style description
- Yearly memory timeline (chronological)
- Bottom: 「和 TA 聊天」button

### Page 3: 物品观察页 (`/observe`)

- Upload image or enter text description
- After submit: show objective description
- Then show each robot's reaction (inner thought + expression), color-coded
- Bottom: 「让它们聊聊这个」button → navigates to chat page and triggers idle chat

### Page 4: 群聊页 (`/chat`)

- Group chat UI: messages with avatar, name, bubble
- Autonomous chat messages appear via SSE with slight delay for typing feel
- User input box at bottom to join conversation
- End state: "对话已结束，已生成新的共同记忆"

### UI Style

Warm, soft, life-like. Tailwind CSS with warm color palette, rounded cards, gentle transitions. "小世界" aesthetic, not tech/sci-fi.

| GET | `/api/conversations/{id}/messages` | Get conversation messages |

## Safety Boundaries (v0.1 Minimal)

Implemented as a post-processing check on all LLM outputs:

- No claiming to be a real human
- No guilt-tripping or possessive language ("I'll be sad if you leave")
- No medical/legal/financial advice
- Crisis detection → de-personalize, suggest professional help

Simple keyword + LLM-based check. No separate safety service in v0.1.
